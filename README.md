# ai-document-extractor

Turn invoice PDFs into clean, validated JSON and Excel.

> **Origin:** generic rebuild of document-extraction tooling I built during an internship at Cypress Industries, where versions of this processed ~2,400 supplier invoices a month into the company's ERP. This repo contains none of that code or data — every sample document is synthetic, generated by `scripts/generate_samples.py`.

## Two engines, one schema

| Engine | How | When |
|---|---|---|
| `text` | Parses the PDF's embedded text layer with patterns. No network, no cost, deterministic. | Digital PDFs with known layouts — the easy majority |
| `llm` | Renders pages to images, Claude vision extracts fields. | Scans, photos, weird layouts |

Both engines emit into the **same pydantic schema**, which validates the arithmetic (line qty×price = amount, lines sum to subtotal, subtotal+tax = total). That's the core design lesson from the production version: extraction output is untrusted input — LLMs and OCR both mis-read numbers, and validation catches it before it reaches a spreadsheet or an ERP.

## Run it

```bash
pip install -r requirements.txt
python scripts/generate_samples.py            # synthetic invoices -> samples/
python -m extractor.cli samples/ --out out/   # text engine (default)
python -m pytest                              # tests, incl. extraction vs ground truth
```

LLM engine (needs an [Anthropic API key](https://console.anthropic.com/)):

```bash
export ANTHROPIC_API_KEY=...
python -m extractor.cli samples/invoice_01.pdf --engine llm --out out/
```

Output: one JSON per invoice plus a combined `invoices.xlsx` (summary sheet + line-items sheet), with a warnings column for anything that failed arithmetic checks.

## What I learned

- Route cheap-and-deterministic first, expensive-and-flexible second — the text engine handles clean PDFs for free and *tells you* when to escalate to the LLM.
- Schema validation is the real interface: prompts change, models change, but the pydantic contract is what downstream systems depend on.
- Ground-truth testing beats eyeballing: samples are generated with known values, so extraction accuracy is a test suite, not a vibe.
